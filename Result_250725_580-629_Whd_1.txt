--- Frage Nr. 609 (Richtig) ---
Frage: Which function is used to unload a relational table into a JSON file?
Deine Antwort:     C
Korrekte Antwort:  C

Erklärung:
The TO_JSON function in Snowflake is used to convert relational data into a JSON-formatted string. It is the appropriate function to use when unloading a relational table into a JSON file, as it serializes the data into a JSON representation.
B. JSON_EXTRACT_PATH_TEXT: This function extracts a specific value from a VARIANT (JSON) object based on a path, returning the value as text. It's used for querying inside existing JSON data.

C. OBJECT_CONSTRUCT: This function is designed to construct an OBJECT (Snowflake's representation of a JSON object) from a list of key-value pairs. When used with SELECT * (i.e., SELECT OBJECT_CONSTRUCT(*) FROM my_table;), it automatically takes all columns from the current row, uses their names as keys, and their values as the corresponding JSON values. This is the most direct and recommended way to convert a relational row into a single JSON object for unloading.

D. TO_JSON: This function converts a VARIANT value (which could already contain a JSON object or array) into its canonical string representation. It serializes an existing VARIANT to a string, it does not construct the VARIANT (JSON object) from individual relational columns.

Therefore, OBJECT_CONSTRUCT is the function you would use in your SELECT statement before the COPY INTO @<stage> command to transform relational rows into JSON objects for unloading.

The final answer is  
C
